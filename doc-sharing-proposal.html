単発の社内共有を「HTMLリンク」にしたい
— やり方をご相談させてください

起案：絵里　／　2026-06-23　／　宛先：BOSS 各位

1ご相談したいこと（主旨）

Claude Code導入指南書や、今回の週次レポート改善提案のような単発の社内共有を、読みやすいHTMLのリンクで共有したいです。やり方として GitHub Pages を試してみたのですが、これで問題ないか／もっといい方法があるか／社内（CCN）に置いた方がよいかを相談させてください。セキュリティや運用ルールはまだこれからです。

2いま「読みにくい」と感じていること

  • Slackに長文テキスト → 流れていくし読みにくい。太字や絵文字の装飾が逆に読みにくく、内容が伝わりにくい。
  • PDF → 内容・使い勝手は良いが、ページ分割の都合や、開く環境（スマホ等）によって読みにくさが出る。

3HTMLが良いと感じた理由

  • 出先でスマホで見ても、PCと同じように読みやすい（画面サイズに自動で合う）。
  • リンク1本で共有でき、開けば常に最新。表・図・色分けもそのまま見える。

※この提案書自体も、まさにそのHTMLリンクで開いていただいています。スマホでも見やすいか試してみてください。

4試してみたこと

GitHub Pages で、Airレジ週次レポートの設置指南書とこの提案書をHTML公開してみました。
→ Airレジ週次レポート 設置指南書（実例）
確認できたこと：今回の置き場所は「GitHub上の公開Web」です。＝URLを知れば誰でも見られる状態で、CCN（社内のClaude Codeネットワーク）の中ではありません。

5手段の整理（私が調べた範囲）

手段スマホ/PCで読みやすい社内限定にできるすぐ使えるか
GitHub Pages（今回試作）◎△ 公開向き（URLを知る人のみ運用）◎ もう動いている
Cloudflare Pages＋Access◎◎ 社内メール限定が可能○ 初期設定が要る
Notion◎◎△ アカウントはあるが全社員が使いこなせておらず研修が必須
CCN（Claude Codeネットワーク）✕ ブラウザ閲覧用ではない◎ 社内クローズド○ ノード間のファイル配布向き
PDF / Slackテキスト△（上記の読みにくさ）◎◎
ざっくり：「スマホでも読みやすいリンク共有」を満たすにはWeb公開（GitHub Pages か Cloudflare）が必要です。CCNはファイル配布の仕組みでブラウザ閲覧には向かず、Notionは展開に研修が要るので今すぐは難しい、という整理です。

6決めさせていただきたいこと

1. 置き場所はGitHub Pagesで問題ないか／もっといい案があるか（公開でよい単発資料を、読みやすくリンク共有する用途）。
2. 公開でよいか／社内限定にすべきか（今回のは公開Web。機微情報を載せるなら Cloudflare Access のような社内限定の仕組みが必要。CCNに置く選択肢もあるが、その場合ブラウザで見やすく開くには別途Web公開が要る）。
3. セキュリティ・運用ルール（公開してよい資料の線引き、誰が整備・管理するか。ここは特に隆志さんに相談したいです）。

まずは公開で問題ない単発資料（指南書・一般的な提案など）からHTMLリンク共有を始めて、機微情報用に社内限定の仕組みを後から足す、という進め方を考えています。ご意見ください。
